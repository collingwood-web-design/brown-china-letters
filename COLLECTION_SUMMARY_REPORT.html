Brown China Letters — Collection Summary Report

Prepared for: Stakeholders Date: 30 May 2026 Status: Transcription complete (source PDFs through 1941)

1. Executive summary

The Brown China Letters project digitizes and transcribes the missionary correspondence of Homer G. Brown and Muriel J. Brown (and family) during their work in West China, spanning roughly 1903 through 1941. The missionaries’ China service ends in 1941; no further letters exist in this collection.

The project converts 83 scanned binder PDFs (~4,270 pages) into:

  1. PNG page images — one image per PDF page, for side-by-side viewing
  2. Markdown transcriptions — one file per letter (or fragment), with structured metadata
  3. Per-year manifests — machine-readable indexes linking each transcript to its scan(s)
MetricCount
Transcribed letters (complete, partial, and lost fragments)2,021

Letter definition for this count: dated correspondence and undated/lost fragments. Excluded from the letter total: cross-binder reference stubs (14), duplicate-scan wiring files (11), bleed-through skip notes (5), photos/clippings (11), and receipts/reports (8).

2. About the correspondence

The collection is primarily letters between:

  • Homer G. Brown and Muriel J. Brown (husband and wife)
  • Mrs. Brown (“Mother”) to children (Muriel, Julia, Isabel, Evelyn, etc.)
  • Family letters during World War I (Homer with the Chinese Labour Corps in France, 1918–1919)
  • Mission life in Chengtu and West China — church work, travel, war years, furloughs, family news

Physical binders use the naming pattern BrownS {binder_year}-v{N}.pdf (e.g. BrownS 1937-v15.pdf). Letters are filed by the date on the letter (content year), not necessarily by which binder holds the scan.

4. Two “years” — content year vs. binder year

Every letter uses two independent date/location concepts:

ConceptMeaningExample
Content yearYear written on the letterLetter dated 15 Mar 1919 → folder transcriptions/1919/
Binder yearWhich physical PDF holds the scanSame letter scanned in BrownS 1918-v11.pdf page 4

Rules:

  • Markdown files live in transcriptions/{date_year}/.
  • source_pdf and source_images always point at the real scan — they do not change when a letter is relocated to the correct content-year folder.
  • When binder year ≠ content year, set source_binder_year so filters and agents know both values.

Example: Jan–Mar 1941 letters to Muriel in Canada were scanned in BrownS 1940-v5.pdf but filed in transcriptions/1941/. The 1940 binder folder holds thin cross-reference stubs so every PDF page is accounted for in audits.

5. Markdown file structure

Each letter is a single .md file with:

  1. YAML frontmatter (metadata between --- delimiters)
  2. ## Transcription — verbatim text of the letter
  3. ## Archivist notes (optional) — fidelity notes, typos preserved, page-boundary comments

Filename convention

{date_year}-{date_month}-{date_day}_{author}_to_{recipient}.md

Examples:

  • 1941-06-02_mother_to_muriel.md
  • 1918-lost-001_homer_to_muriel.md (undated fragment)
  • 1941-odds-001_elson_chengtu_district_fund_receipt.md (non-letter document)

6. How PNGs are extracted from PDFs

Source PDFs are scanned images (no text layer). Each page is rendered to PNG at 2× zoom for legibility.

Command:

cd Original_Letters
python scripts/render_pdf_pages.py --year 1937

Output path pattern:

pages/{binder_year}-v{volume}/page_{NN}.png
PDFPNG folderPage 1 image
BrownS 1941-v6.pdfpages/1941-v6/pages/1941-v6/page_01.png
BrownS 1937-v15.pdfpages/1937-v15/pages/1937-v15/page_01.png

Page numbers are zero-padded to two digits (page_01, not page_1).

7. How transcripts attach to PNGs

The link between transcription and scan is explicit in every letter file. No guessing — the website and vector store read these fields directly.

Wiring rule

source_pdf_pages[N]  →  source_images[N]

Index N pairs the PDF page number with the matching PNG path.

source_pdf_pagessource_images
[17]pages/1941-v6/page_17.png
[1, 2]pages/1940-v5/page_01.png, pages/1940-v5/page_02.png

Cross-volume letters

Some letters span two binder PDFs. Page numbers in source_pdf_pages are relative to source_pdf for the primary volume; additional pages appear in source_images with paths pointing at the other volume:

source_pdf: BrownS 1918-v5.pdf
source_pdf_pages: [22, 1, 2]
source_images:
  - pages/1918-v5/page_22.png
  - pages/1918-v6/page_01.png
  - pages/1918-v6/page_02.png
reading_order: [22, 1, 2]

Side-by-side viewer

When side_by_side.enabled: true, the UI loads each PNG in source_images alongside the markdown body under ## Transcription. Paths are relative to Original_Letters/.

Data flow (overview)

BrownS YYYY-vN.pdf → render_pdf_pages.py → pages/YYYY-vN/page_NN.png → transcriptions/YYYY/*.md → build_manifest.py → manifest.json → Website / vector store

8. Manifest files (manifest.json)

Each content-year folder has a manifest.json built from letter frontmatter. It is the machine-readable index for the website and search ingestion.

Location: transcriptions/YYYY/manifest.json

Sample entry:

{
  "letter_id": "1941-01-05-homer-muriel-001",
  "letter_status": "complete",
  "date": "1941-01-05",
  "author": "Homer G. Brown",
  "recipient": "Muriel",
  "markdown": "transcriptions/1941/1941-01-05_homer_to_muriel.md",
  "source_pdf": "BrownS 1940-v5.pdf",
  "source_binder_year": "1940",
  "source_pdf_pages": [1, 2],
  "source_images": [
    "pages/1940-v5/page_01.png",
    "pages/1940-v5/page_02.png"
  ]
}

Build / validate:

python scripts/build_manifest.py --year 1941

This verifies every source_images path exists on disk.
